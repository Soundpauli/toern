Menu map

The whole settings tree at a glance — every leaf checked against firmware. Blue rows are root families; red rows are nested leaves. This is the navigation map; per-screen previews live in the screen index.

Root menu — how navigation works

Open the menu by holding a touch pad from the grid. The root list uses red and blue dots on row 16; the encoder 4 hint strip is green to signal "this knob navigates". Submenu families re-colour the same row with the parent menu colour, so you always know which world of settings you are inside.

The full tree

Ten root families. Short labels are exactly what the matrix prints. Indentation is how deep you have dived. Every leaf below was checked against the firmware — if the device can do it, it is here; if it cannot, it is not.

Why blue vs red?

MENU Root of the map (this page’s “you are here”).
FILE Blue tint = depth 1 — a root-menu family you enter with encoder 4. On the device these are the coloured dots on row 16.
MAIN Red tint = depth 2 — a leaf inside a family (VOL, SETTINGS, REC, MIDI, ETC). Blue badge, warmer row: you are one click deeper.

The colours are a handbook nesting cue — not the device’s voice palette. On hardware, submenu text re-colours to match its parent family so you always know which world you are inside.

  • MENU Hold a front touch pad from the grid. Playback keeps running — the menu is a translucent overlay, not a pause button. Encoder 4 spins the ten family dots on row 16; short-press enters. Encoder 1 short steps back. Re-open within five seconds and you land on the last family; wait longer and you start on FILE again (the firmware’s polite “where were we?”).
    • FILE Pattern vault. One slot = the whole session on the card: all 16 pages, every voice, every per-step value, every tone knob — everything except which sample pack is loaded.
      • Enc 1 short — load
      • Enc 2 short — save (overwrites, no “are you sure?”)
      • Enc 3 short — sidecar on/off (when on, load also pulls the matching .txt: palette, MIDI offsets, pack hint)
      • Enc 4 turn — slot 1–99 (bright green = occupied, dim blue = empty)
      • Enc 4 short — back to the grid
       Try to load an empty (blue) slot and you do not get an error — you get NEW, the pattern generator. That is intentional: empty file → invent something. Save first if the current RAM pattern still matters.
      • NEW Genre pattern generator — not a root menu item. Reached only by loading an empty FILE slot. Seeds a fresh pattern into RAM, then drops you on the grid.
        • Enc 1 short — abort back to FILE
        • Enc 3 turn — length 1–16 pages (disabled for BLNK — blank is blank)
        • Enc 4 turn — genre: BLNK (wipe clean), TECH, HIPH, DNB, HOUS, AMBT
        • Enc 4 short — generate & load
        Same settings → different roll every time; the dice have feelings.
    • PACK Which numbered folder of WAVs fills sample voices 1–8. Synths 11 / 13 / 14 ignore this; rows 9, 10, 12 stay empty forever (by design, not by bug).
      • Enc 1 short — load pack (replaces all eight voices)
      • Enc 2 short — remember this pack across reboots
      • Enc 4 turn — pack 0–99 (green = folder exists, blue = missing). Pack 0 is the scratchpad for one-off browser loads
      • Enc 4 short — back
    • WAVE Sample browser for voices 1–8. Also opens with a both-touch (left-first) from the grid on a sample row. On a synth / empty row the root entry shows (−) and refuses to open — no WAV for a voice that has no sample engine.
      • Enc 1 turn — trim start 0–100 %
      • Enc 1 short — directory up
      • Enc 2 turn — trim end (always above start)
      • Enc 2 short — reverse preview
      • Enc 3 short — load trimmed file + autosave
      • Enc 4 turn — browse files
      • Enc 4 short — enter / leave folder
      • Touch 3 long — jump into record
    • BPM Tempo + who owns the clock. Colour gossip: cyan digits + green right-arrow = you are master (INT); green digits + green left-arrow = external locked; red + red left-arrow = still hunting or the cable is lying.
      • Enc 2 turn — matrix brightness 3–255 (pretty, not loud)
      • Enc 2 short — toggle idle tint
      • Enc 3 turn — INT ↔ EXT (survives reboot)
      • Enc 4 turn — BPM 40–300 in INT; in EXT only nudges a stored offset
      • Touch pads — tap-tempo on this screen
    • VOL Where sound leaves the box. Six leaves. Encoder 4 walks siblings; encoder 3 usually turns the value (watch the lit cue).
      • MAIN Headphone level on the 6.35 mm jack.
        • Enc 3 turn — 0–100
        • Quiet rehearsal → couch hero → neighbour complaint. Orange-ish digits climb with the level
      • LOUT Line-out trim, independent of MAIN — phones loud, mixer calm.
        • Enc 3 turn — display 1–19 (maps to the codec’s 13–31 under the hood)
        • If the desk is clipping, lower this before you blame the kick
      • PREV How loud the sample browser auditions files.
        • Enc 3 turn — 0–50
        • Zero = silent browsing (blessed on stage). Independent of MAIN/LOUT
      • 2-CH Stereo personality of the eight sample voices.
        • OFF — mono centre
        • M+P — mono plus a stereo “perception” pad
        • L+R — real width via per-voice pan
        Mono PA? OFF. Stereo desk? L+R.
      • SPKR Built-in speaker.
        • ON — hear through the built-in driver
        • OFF — mute the speaker (line out stays live)
        Mute the speaker for dark stages, not the gig.
      • HFC Global high-frequency cut (GEQ treble band) for bright cans or long sessions.
        • Enc 3 turn — 0, then 10…250 in tens, then 256 (max / flat-ish endpoint)
        • Lower = softer top. Hits every output the same
    • SETTINGS How the grid behaves — visuals, page walking, paint style. Zero effect on the audio engine; pure “feel”.
      • FLW Follow — whole-grid pulse on the beat.
        • ON — soft wash in time (club-dark friendly)
        • OFF — only the playhead column (editor-clean)
      • PREV Sample-browser preview policy (not the VOL→PREV loudness).
        • ON — auto-audition while you spin files
        • PRSS — only when you press (so a clumsy turn does not scream into the PA)
      • VIEW How densely notes paint on the grid.
        • EASY — current page only
        • FULL — current page bright, neighbours ghosted underneath for context
      • PMD Pattern mode — what happens when a page finishes.
        • OFF — loop this page forever
        • ON — advance to the next page that has notes
        • SONG — follow the SONG chain
        • NEXT — jump once to a queued page at loop end (live transitions)
      • LOOP How many steps of a page actually sound before it wraps.
        • OFF — full 16 steps
        • 1–16 — chop short
        Odd lengths (5, 7, 9…) = instant groovebox weirdness.
      • CTRL What encoder 2 does on the draw/single grid when you are not in a special mode.
        • PAGE — flip pages 1–16
        • VOL — per-row volume for the cursor’s lane (live mix fader)
        Survives reboot.
      • LEDS Which LED panel hardware is bolted on.
        • 1 — one panel
        • 1B — one mirrored
        • 2 — two stacked (16×32)
        • 2B — two mirrored
        Wrong setting = pixels vacationing off-screen.
      • PONG A ball bounces the grid in draw mode; every time it hits a cell that already has a note, that note fires again.
        • OFF / ON
        • Enc 3 turn — speed 1–99 when on (higher = faster)
        Not the idle “eyes” screensaver — that is a separate boredom animation.
      • CRSR Cursor cosmetics.
        • NORM — thin white ring
        • CHNR — only on the focused voice in single mode
        • BIG — chunky 3×3 halo for balcony viewers
      • DRAW Paint-handedness.
        • L+R — set with enc 4, remove with enc 1
        • R — enc 4 toggles set/remove; enc 1 = break mute
        Careful two-hand edit vs one-hand set/remove. Detail: Sequencing → triggers.
      • MUTE Custom unmute mask for leaving R-mode’s temporary full mute (enc 3).
        • Enc 1 turn — select channel 1–16
        • Enc 2 turn — toggle that channel in / out of the unmute mask
        • Green — comes back unmuted
        • Red — stays muted
        Build a “kick+snare only” return set. Does not gate which rows you can paint.
    • REC Capture path: which jack, how hot, when it rolls, and what happens to notes after a re-record.
      • INPT Recorder input.
        • MIC — mic path
        • LINE — top-edge line in
        Flips which gain page (MIC vs L-IN) actually matters.
      • MIC Mic preamp gain.
        • Enc 3 turn — 0–63
        • Watch the record meter; red peaks mean “back off, hero.”
      • L-IN Line-in trim (coarser — line gear is already loud).
        • Enc 3 turn — 0–15
        • Synths often happy around 6–10
      • TRIG When capture actually starts.
        • OFF — manual
        • SENS — threshold (hands-free acoustic sampling)
        • −CON / +CON — count-in flavours
      • CLR What happens to notes on the voice you just re-recorded into.
        • OFF — keep notes (new sound under old arrangement)
        • ON — wipe the voice
        • FIX — wipe notes but keep vel / prob / lane vol
        • ON1 — wipe current page only
        • CLIC — wipe and plant a click on step 1 so you can hear the sample on the grid
    • MIDI TRS MIDI in/out plus an optional analog PPQN pulse for modular / vintage friends.
      • CH How incoming notes map onto voices.
        • YPOS — MIDI ch N → row N
        • MIDI — notes hit the currently selected voice
        • KEYS — pitch drives synth notes; samples trigger from velocity
      • TRAN Start / stop / continue.
        • OFF — ignore & never send
        • GET — follow an external sequencer
        • SEND — make others follow you
        Pair with BPM’s INT/EXT for full master/slave.
      • SEND What leaves the MIDI out jack.
        • CLCK — clock only
        • NOTE — notes only
        • BOTH — clock + notes
        No separate OFF here — mute notes via RCVE / stop sending clock via BPM EXT as needed.
      • RCVE Do incoming notes poke the internal engines?
        • NOTE — yes, per CH mapping
        • OFF — ignore notes (pure sequencer/recorder mode)
      • SYNC Latency makeup. Two lines: SNC1 send path, SNC2 receive path.
        • Enc 2 turn — pick SNC1 / SNC2
        • Enc 3 turn — offset −127…+127 ms
        Nudge until the flam vanishes.
      • PPQN Analog clock pulse on Teensy pin 31 (J11 / I2SC_3V), independent of TRS MIDI.
        • Enc 2 turn — polarity + / −
        • Enc 3 turn — OFF or 1, 2, 4, 8, 12, 16, 24, 32 PPQN
        Follows INT BPM or EXT MIDI clock. DIY jack: white → tip, blue → sleeve — see MIDI → PPQN.
    • SONG 64-slot arranger. Each slot points at one of the 16 pages. With PMD = SONG, playback walks the chain and wraps.
      • Enc 2 turn — page 1–16 (or empty)
      • Enc 2 short — clear slot
      • Enc 4 turn — slot 1–64
      • Enc 4 short — insert empty slot
      Empty slots are skipped. Playing: page strip on row 16 + green song-slot dot on row 1.
    • ETC The junk drawer that somehow contains the cool stuff: credits, SD file server, AI fill, ring light, palette, battery, child lock, nuclear resets.
      • INFO Firmware version, then a scrolling thank-you.
        • Read-only
        • Version holds in yellow for two seconds, then the credits crawl
        Confirm the build before filing a bug report.
      • SD USB serial file server — stay on this page while a computer talks to the card.
        • WAIT — until a host connects
        • OK — connected
        • Use sdtool.tyng.app (Chrome/Edge + Web Serial)
        Reopen after Connect if the unit reboots. Details: Hardware → SD tool.
      • AUTO Generative fill: learn a base page range, then invent the pages after it in the same style. Orange strip = source pages; green strip = pages about to be written.
        • Enc 1 turn — how many extra pages to generate (1–16)
        • Enc 1 short — run generation (green cue — the rare “press 1 to create” exception)
        • Enc 2 turn — base start page
        • Enc 3 turn — base end page
        Same settings → fresh dice every time.
      • LGHT Optional chassis ring-LED strip.
        • ON — mirrors the matrix accent
        • OFF — saves juice / keeps the stage dark
      • COLR Voice palette / scheme.
        • 0 — factory rainbow
        • 1 — blue-ish
        • 2 — high-contrast red/violet
        • Custom fourth slot appears when you drop a scheme.txt from the colour-scheme tool
      • BATT State of charge.
        • Read-only %
        • Green — > 30 %
        • Orange — 15–30 %
        • Red — < 15 %
        Plug USB-C before the red argument starts.
      • CHLD Child lock — simplified play mode for small hands (or reckless bandmates).
        • OFF / ON
        • When on: menu opens only via touch 2 then touch 1 within two seconds
        • Editing stays on page 1
        • Synth voices are muted
        • Filter / velocity / subpattern screens are blocked
        Training wheels with opinions.
      • RSET Five rungs of increasingly spicy recovery. Pick with encoder 3, confirm with encoder 4 — then wait for DONE.
        • SD — rescan the card (safe; useful after swapping files)
        • EFX — factory-reset every tone control (patterns & sample assignments survive)
        • FULL — blank slate + SD rescan (BPM→100; only FILE-saved work comes back)
        • FILE — wipe saved patterns 1–99.txt (keeps autosave)
        • PACK — delete pack folders except pack 1
        FILE and PACK are the “I meant it” buttons.
